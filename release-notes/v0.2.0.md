# Codex Provider History Fixer v0.2.0

本次版本重点修复迁移安全性、运行稳定性和 Windows 打包后的 native SQLite 兼容问题。

## 新增

- 迁移前检测 Codex CLI 和 Codex App 进程，未关闭时阻止继续写入。
- 检测到 Codex 正在运行时，提供重新检测入口。
- 新增“一键关闭全部 Codex”按钮，并在执行前进行二次风险确认。
- 迁移任务改为在后台 worker 中执行，降低备份和写入期间界面卡死或无响应的概率。

## 修复

- `state_5.sqlite` 写入改为使用原生 SQLite 事务更新，不再用 `sql.js` 导出整库后覆盖文件。
- 修复 Electron 打包后 `better-sqlite3` ABI 不匹配导致的启动或迁移失败。
- 修复打包后 worker 无法找到 `core/historyStore.js` 的问题。
- 修复打包后 `better-sqlite3` 无法找到 `bindings` 运行依赖的问题。

## 安全说明

- “一键关闭全部 Codex”只会结束检测到的 Codex 进程，不会删除本地文件或数据库。
- 强制关闭可能中断正在运行的 Codex 任务，因此执行前会明确提醒用户确认。
- 迁移前仍会创建完整备份，包括 `state_5.sqlite`、WAL/SHM 文件和 `sessions` 目录。
